Quirquincho

The chaucha guardian

Introducción

Quirquincho es un Bot que permite conectar la Red Chaucha con el grupo de Telegram del Proyecto Chaucha, y fue nombrado en honor al armadillo peludo que habita el altiplano de la Cordillera de los Andes.

Este proyecto nació de manera accidental, en base a una pelea interna entre los desarrolladores del Proyecto Chaucha para decidir la manera correcta de diseñar un Bot para la comunidad.

Listado de comandos

La comunicación entre el usuario y Quirquincho se realiza a partir de comandos específicos para cada tarea a desempeñar, que pueden ser ejecutados de manera privada, hablando directamente con @QuirquinchoBot, o de manera pública dentro del grupo de Proyecto Chaucha.

Cada comando ejecuta una tarea especifica dentro de la Red Chaucha y entrega un mensaje de respuesta en formato de texto.

A continuación se presenta la lista completa de comandos que interpreta Quirquincho, junto con una pequeña explicación de su funcionamiento, la sintaxis para ejecutar correctamente el comando y un ejemplo simple de pregunta y respuesta.


Online Wallet

/balance
Saldo disponible de Chauchas y address que pertenece al usuario que ejecutó el comando.

Modo de uso:
/balance

Respuesta:
Tienes {TOTAL} CHA en tu dirección
{CONFIRMADO} CHA para utilizar y {PROCESANDO} CHA sin confirmar.

Tu address es {ADDRESS}


/send
Enviar Chauchas especificando el monto a transferir y la dirección de destino.

Modo de uso:
/send {cantidad} {address}

Respuesta:
insight.chaucha.cl/tx/{TXID}


/sendall
Enviar todas las Chauchas que mantiene la dirección del usuario hacia una dirección de destino.

Modo de uso:
/sendall {address}

Respuesta:
insight.chaucha.cl/tx/{TXID}


/qr
Creación de código QR con la dirección asignada al usuario.

Modo de uso:
/qr

Respuesta:
{IMAGEN QRCODE}


Información

/op_return
Almacenar un mensaje en el blockchain de Chaucha.

Modo de uso:
/op_return {mensaje}

Respuesta:
insight.chaucha.cl/tx/{TXID}


/mensajes
Rescatar mensajes almacenados con la utilización del comando /op_return

Modo de uso:
/mensajes

Respuesta:
{FECHA 1}: {MENSAJE 1}
{FECHA 2}: {MENSAJE 2}
...
{FECHA N}: {MENSAJE N}
